  • Two shapes of edit, two depths. A short value edits in the row — `was info ·` stays visible while you type. A multi-line key (`config.yaml`) opens the 17a buffer editor scoped to just that value, with the same dry-run-first apply. Both are value-edits: the key exists, only `data.` changes.
  • The consumers strip is the point. Applying a configmap change fixes nothing until pods re-read it — the strip names every workload that mounts or envs this map (from the watch), and `ctrl-r` chains the apply with `kubectl rollout restart` for each, both commands printed. Plain `↵` applies without restarting; kute never restarts on its own.
  • Conflict handling matches 17a: the patch carries the observed resourceVersion; if the map changed underneath, the banner offers diff / rebase / discard.
