  • The rule is a class, not a judgment call: a verb renders in the bar iff its registry entry is kind-scoped. Global grammar (goto, filter, mark, yaml, edit, events, timeline, meta, scopes) is by definition the same everywhere — a legend that repeats it on every screen is teaching you what you already know, at the cost of the keys you came for.
  • Nothing is removed from the app — every retired key keeps working and stays in the 7b overlay, whose Scope + Global columns were built for exactly this. The keybar's right slot renames from `? help` to `? all keys` so the contract is explicit.
  • Labels compress with the count: `rollout restart` → `restart`, `pods of deployment` → `pods` — at six groups each label can be one word and still be unambiguous.
  • Fails if a global key is genuinely undiscoverable — the mitigation is 29c's teaching layer, which composes with this rule.

  • Rank is data on the registry entry: kind-scoped verbs first, then globals by measured session frequency (`/` and `y` earn slots on wide terminals; `a`, `c`, `t` never do). The bar fills rank-order until the next group wouldn't fit, then renders `? … N more`. One renderer, every width — no hand-tuned per-kind bars.
  • The overflow slot is a contract, not an ellipsis: N is the exact count of live, applicable verbs not shown, and `?` opens 7b with those N sorted to the top. Nothing silently disappears.
  • Degrades exactly like the tables do at narrow widths — same philosophy as the min-size guard: show less, never lie.
  • Weakness: rank-by-frequency means the bar can differ subtly between screens of the same kind class, and between users once frequency is personal — 29a's class rule is more predictable; this is the choice if predictability loses to utility.

SAME USER, DAY 1 — THE FULL TEACHING BAR
DEPLOY ↵ pods i image +/− scale R restart f forward │ ctrl-d delete ? all keys
  • The keybar's real job is teaching, and teaching ends. A verb you've fired ≥15 times over ≥3 sessions retires from the bar — you've proven you don't read it. ↵ and R retired here; `i` stays with a `new in 0.2` tag because it shipped after your counts, `f` because you've never pressed it. The bar converges toward empty for experts and stays full for newcomers, with zero configuration.
  • Retired ≠ gone: every key works forever and `? all keys` is the complete map (7b). Counts live in the versioned per-user state (same file as recents), so the schema bump + migration rule applies; `keybar.teach: false` pins the day-1 bar for people who want a permanent legend.
  • Destructive verbs never retire — `ctrl-d` disappears only under 29a's class rule, never by learning; you don't want the delete affordance to be a memory test. (Shown composed with 29a: globals are already gone on day 1.)
  • Weakness: the bar becomes non-deterministic across machines and screenshots — docs and pairing suffer. If that kills it, 29a alone is the fallback; the `new in 0.2` tag is worth keeping regardless.
